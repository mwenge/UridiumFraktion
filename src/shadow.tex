Like the structure's that litter the surface of the dreadnought, the player's manta ship
casts a shadow below it. While it is a simple effect, it is an important one for creating
the impression of depth and solidity during gameplay. Careful attention has been paid
to the positioning of the shadow. Observe below how it is consistent with the shadows cast
by features on the surface. They all share a common light source emanating from the top left
of the screen, at an angle of approximately 45 degrees.

\begin{figure}[H]
    \begin{adjustbox}{width=13.2cm,left}
      \includegraphics{src/shadow/level1.png}%
    \end{adjustbox}
\end{figure}
\vspace{-0.4cm}

The exact positioning of the shadow is 7 pixels down and 14 pixels to the right. Notice how 
shadows of the structures on the dreadnought obey the same down by 1 and over by 2 formula: 
\begin{figure}[H]
    \begin{adjustbox}{width=13.2cm,left}
      \includegraphics{src/shadow/level7.png}%
    \end{adjustbox}
\end{figure}
\vspace{-0.4cm}
This effect is drawn rather economonically inside a clause called \icode{DrawManta\-AnimationFrame}
nested inside the routine for animating the Manta ship. The procedure consists of drawing the Manta
ship first, followed by drawing the shadow at an X and Y offset to the ship's position given by
\icode{mantaShadowOffset}. The Y offset is obtained by dividing \icode{mantaShadowOffset} by 2. No division
is necessary for the X offset.

\begin{lstlisting}
DrawMantaAnimationFrame   
  LDA #$06                    ; Load the Manta's index to A
  STA spriteIndex             ; Store it in spriteIndex
  JSR GetCurrentSprite        ; Get the Manta's sprite parameters.
  LDA newSpriteValue          ; Load the new sprite for Manta to A.
  STA currentSpriteValue      ; Store it in currentSpriteValue.
  ADC #$30                    ; Add 48 to get the shadow sprite.
  STA mantaShadowSpriteValue  ; Store it in mantaShadowSpriteValue.
  LDA mantaCurrentYPos        ; Load the current Y position to A.
  STA currentSpriteYPos       ; Update the sprite's Y position.
  JSR DisplayCurrentSprite    ; Display the Manta sprite.

  ; Draw the Manta's shadow.
  INC spriteIndex             ; Increment the index to point to the shadow.
  JSR GetCurrentSprite        ; Get the shadow's sprite parameters.
  LDA mantaShadowSpriteValue  ; Load the shadow sprite value to A.
  STA currentSpriteValue      ; Store it in currentSpriteValue.
  LDA mantaShadowOffset       ; Load the offset for shadow to A.
  LSR                         ; Divide by 2.
  ADC mantaCurrentYPos        ; Add it to Y position.
  STA currentSpriteYPos       ; Store as the shadow Y position.
  LDA #MANTA_HORIZONTAL_POS   ; Load the manta X position to A.
  ADC mantaShadowOffset       ; Add the offset for shadow.
  STA currentSpriteXPos       ; Update the shadow's X position.
  JSR DisplayCurrentSprite    ; Display the shadow sprite.
\end{lstlisting}

An alert player will notice that the shadow is clipped (as we would expect) when the Manta
is over the dreadnought's edge. Somehow, when we draw the shadow in this instance, the
shadow sprite is not drawn 'over' the background, but is 'underneath' it, or perhaps not
drawn at all.
\begin{figure}[H]
    \begin{adjustbox}{width=13.2cm,left}
      \includegraphics{src/shadow/level9_clipped.png}%
    \end{adjustbox}
\end{figure}
\vspace{-0.4cm}
How is this achieved we might wonder? The answer lies in a rule which the C64 will apply to
sprites that are configured to display 'behind a backgound'. A background in this case is any
character on the screen where the pixel pair begins with a 1. Since our background, which is the
inky blackness of space, is rendered using a pair of \icode{1} bits (character \icode{\$20}) as illustrated
in the diagram below, any sprite we configure
this way will get hidden behind the background rather than displayed above it. 

\begin{wrapfigure}[9]{r}{6cm}
    \vspace{-0.2cm}
    \begin{adjustbox}{width=6cm,left}
      \includegraphics{src/character_set_diagrams/9_20.png}%
    \end{adjustbox}
\end{wrapfigure}

This configuration bit, which is in register \icode{\$D01B}, is set only on the sprite used to display the shadow. 
The sprites used for everything else: the manta, its
bullets, and the meanies we spend most of time our shooting at do not have this bit set so will always
display above the background, as expected. Through careful selection of initial values for the background we've
avoided the need for any elaborate trickery and achieved a neat little effect that reinforces the illusion of depth.

The final noteworthy use of the shadow is when the Manta is landing on the dreadnought's deck at the end of 
each level. We want to create the illusion of the Manta lowering onto the surface. Strictly speaking, this would require
us to make the Manta gradually smaller as it moves further away from the perspective of the player onto the deck. There is
no need for this kind of complexity though. Instead, it will suffice to use the shadow to achieve the desired effect. We have
already seen how \icode{mantaShadowOffset} is used to calculate the horizontal and vertical position of the shadow in relation
to the Manta. If we gradually reduce this offset from its initial value of \icode{19} to some lower value, and recalculate the
position of the shadow at each turn, we can create the impression that the player's ship is slowly lowering to the surface.

The code that implements this is compact enough to reproduce in full. It is a tight loop that updates and displays
the Manta and its shadow at each turn. We use a frame rate variable to limit the speed at which the Manta is lowered
onto the deck. When we are ready to lower another notch we decrement \icode{mantaShadowOffset} once more and keep looping
until the offset has reached a value of \icode{6}.
\begin{lstlisting}
MantaLandingLoop   
  JSR ProcessGameFrame    ; Process a frame of game time.
  JSR AnimateMantaShip    ; Update the Manta.
  JSR UpdateSprites       ; Update the other sprites.
  LDA frameRate           ; Check the frames rendered.
  AND #$03                ; If less then 7 ..
  BNE MantaLandingLoop    ; .. then display again.
  DEC mantaShadowOffset   ; Decrement the shadow offset.
  LDA mantaShadowOffset   ; Check the new value of the offset..
  CMP #$05                ; If it's greater than 5..
  BCS MantaLandingLoop    ; .. then move the shadow.
\end{lstlisting}

If we set out the appearance of the manta and its shadow at each iteration of this loop, along with the value of 
\icode{mantaShadowOffset}, we can see how the effect is achieved. One thing worth noting is that decrementing
\icode{mantaShadowOffset} results in sort of lockstep progress where the shadow moves first left and then up. This
is because the vertical position is calculated by dividing \icode{mantaShadowOffset} by 2: for odd values of
\icode{mantaShadowOffset} this will result in a clamped value that is the same as the previous even value of
\icode{mantaShadowOffset}.
